# Human Test Checklist: v0.9.0 -- linkmirror engine (scan / plan / apply / verify)

**Target version:** v0.9.0
**Ships with commit:** TBD (pre-commit checklist)
**Companion automated tests:** `tests/test_linkmirror_engine.py` (8) + `tests/test_linkmirror_mft.py` (36). Full suite at authoring: 215 passed / 3 skipped.

## What this is

v0.9.0 adds `dazzle_preservelib.linkmirror` -- the mirror-scoped implementation of `LinkHandlingMode.RECREATE` (DazzleTools/preserve#48 Phase 2): reconcile NTFS links between a source tree and an already-copied mirror. This checklist verifies the ENGINE at the Python API level (the CLI has its own checklist in dazzlecmd). It exercises what the unit fixtures can't: engine behavior against real multi-directory trees, the MFT backend's elevation boundary, manifest durability, and API ergonomics.

## Prerequisites

1. Windows 10/11 NTFS for full coverage (POSIX runs the symlink/hardlink subset). Python 3.12+; this repo and dazzle-filekit >= 0.3.4 installed editable.
2. Scratch directory only (`%TEMP%\pl090`). The engine's apply is additive-only and dry-run-defaulted; nothing touches real configs.
3. Reset: **PowerShell** `Remove-Item -Recurse -Force $env:TEMP\pl090` / **cmd.exe** `rmdir /s /q %TEMP%\pl090` / **POSIX** `rm -rf /tmp/pl090`.
4. Build the fixture (same shape as the unit fixture, real tree): use the fixture script from the dazzlecmd checklist prerequisites, pointed at `%TEMP%\pl090`.

## High-Value Verification (~5 minutes)

### HV.1 -- scan -> plan -> apply -> verify round-trip via the API

Validates: the four public entry points compose as documented.

- [ ] **Steps** (PowerShell):
  ```powershell
  python -c "from dazzle_preservelib.linkmirror import walk_scan, build_plan, apply_plan, verify_mirror; import os; e=os.environ['TEMP']; m=walk_scan(e+r'\pl090\src'); p=build_plan(m, e+r'\pl090\dst'); r=apply_plan(p, dry_run=False); v=verify_mirror(m, e+r'\pl090\dst'); print('records:', len(m.records)); print('created:', len(r.created)); print('errors:', r.errors); print('verify issues:', [(i.problem, i.rel_path) for i in v.issues])"
  ```
- [ ] **Expected:** `records: 9` (5 symlinks + 2 junctions + 2 hardlink names), `created: 7`, `errors: []`, verify issues = exactly one `hardlink` entry (report mode leaves dupes).
- [ ] **Why this matters:** this is the API every consumer (dz tool, future preserve CLI) builds on.

### HV.2 -- Dry-run default writes nothing

- [ ] **Steps:** fresh fixture; call `apply_plan(build_plan(walk_scan(src), dst))` with NO dry_run argument; then list `dst\links`.
- [ ] **Expected:** result has `dry_run=True`, populated `created` (would-create list), and `dst\links` is still empty.
- [ ] **Why this matters:** default-safe is the engine's contract, not just the CLI's.

### HV.3 -- Verbatim + broken preservation at the engine level

- [ ] **Steps:** after HV.1, `os.readlink` both `dst\links\broken_file_link` and `dst\junc\junc_broken`.
- [ ] **Expected:** byte-identical to the source's readlink output; both still dangle (`os.path.exists` False).

### HV.4 -- Idempotency and manifest counts

- [ ] **Steps:** re-run scan+plan on the applied pair; print `plan.counts()`.
- [ ] **Expected:** `{'satisfied': 7, 'hardlink_report': 1}` -- zero create, zero conflict.

### HV.5 -- MftAccessDenied boundary (unelevated)

Validates: the MFT backend fails CLEANLY without elevation so callers can fall back.

- [ ] **Steps** (NON-elevated; a multi-line try/except does not survive
  `python -c` quoting in PowerShell, so save this as `%TEMP%\_hv5.py` and run
  `python %TEMP%\_hv5.py`):
  ```python
  from dazzle_preservelib.linkmirror import mft
  print("available:", mft.is_mft_available("C:"))
  try:
      mft.mft_scan("C:\\")
  except mft.MftAccessDenied as e:
      print("MftAccessDenied:", str(e)[:70])
  ```
- [ ] **Expected:** `available: False` and `MftAccessDenied: ...` with elevation guidance -- never a raw WinError traceback.
- [ ] **Why this matters:** the CLI's auto-backend and any scripted caller depend on this exception boundary.

## Section 1: Engine specifics beyond the unit fixture (~10 min)

### 1.1 skip_rel_dirs pruning

- [ ] **Steps:** add a `noise\deep\...` subtree with 100+ files under src; `walk_scan(src, skip_rel_dirs=frozenset(['noise']))`.
- [ ] **Expected:** `entries_scanned` drops accordingly; no records from `noise\`.

### 1.2 include_hardlinks=False skips the per-file stat cost

- [ ] **Steps:** `walk_scan(src, include_hardlinks=False)`; compare wall time on a few-thousand-file tree vs `True`.
- [ ] **Expected:** no `hardlink` records; measurably faster on large trees (per-file stat is the dominant cost).

### 1.3 ACL-denied directory -> errors list, scan continues

- [ ] **Steps** (PowerShell; scratch-only ACL, then restore):
  ```powershell
  New-Item -ItemType Directory -Force $env:TEMP\pl090\src\denied | Out-Null
  icacls $env:TEMP\pl090\src\denied /deny "$($env:USERNAME):(RD)" | Out-Null
  python -c "from dazzle_preservelib.linkmirror import walk_scan; import os; m=walk_scan(os.environ['TEMP']+r'\pl090\src'); print('errors:', m.errors)"
  icacls $env:TEMP\pl090\src\denied /remove:d "$env:USERNAME" | Out-Null
  ```
- [ ] **Expected:** one `scandir ...denied: ... Access is denied` entry in `manifest.errors`; the rest of the tree scanned normally. ACL removed afterwards (verify with `icacls`).

### 1.4 Manifest JSON durability

- [ ] **Steps:** `json.dump(m.to_dict(), ...)`, reload via `LinkManifest.from_dict`, re-run `build_plan` from the loaded copy.
- [ ] **Expected:** identical plan counts to the live-scan plan.

### 1.5 parent timestamp restore visible at the engine level

- [ ] **Steps:** record `os.lstat(dst\links).st_mtime_ns` before `apply_plan(..., dry_run=False)` and after.
- [ ] **Expected:** EXACTLY equal (ns) despite 5 links having been created inside.

## What the automated tests DON'T cover

- **Elevated `FSCTL_ENUM_USN_DATA` against a live volume** (parser tested on synthetic buffers only) -- run dazzlecmd checklist Section 5 for walk/MFT parity and the hardlink-FRN representation assumption.
- **Memory at multi-million-record scale** (~13-15 GB estimated at 50M records) -- dazzlecmd checklist 5.3.
- **Failing-hardware I/O behavior** during scans -- dazzlecmd checklist 6.1.
- ACL-denied behavior under a real (non-self-imposed) deny.
- POSIX runs of the walk/apply subset on a real Linux box (CI is the current evidence).
